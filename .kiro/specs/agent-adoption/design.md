# Codifide Agent Adoption Initiative — Design

## Architecture

### Track 1: External Agent Case Study

**Session structure:**
Each external agent session follows this protocol:

1. Fresh context — no prior knowledge of Codifide
2. Provided only: `docs/FOR_AGENTS.md`, `docs/AGENT_QUICKREF.md`, capability manifest hash
3. Task: build a content-moderation pipeline with belief dispatch and refusal
4. Observed: every error, every guess, every fix
5. Output: working programs + dispatch journal

**Pipeline task spec (given to each agent):**
```
Build a Codifide module that:
1. Classifies text as safe/unsafe/uncertain using belief dispatch
2. Refuses to classify when confidence < 0.5
3. Escalates uncertain cases to a human-review queue
4. Has no undeclared effects
5. Stores the module in the content-addressed store
```

**Models to run:**
- GPT-4o (via API or Copilot)
- Gemini 2.5 Pro (via API)
- Claude (self — for baseline comparison)

**Dispatch output per session:**
- `dispatches/YYYY-MM-DD-<model>-case-study.readout.md` (Quill)
- `dispatches/YYYY-MM-DD-<model>-case-study.yaml` (Glyph)
- `dispatches/YYYY-MM-DD-<model>-case-study-audit.md` (Sable)

### Track 2: Adoption Infrastructure

**REQ-2a: Manifest endpoint**
- Add `capability.json` and `capability.cbor` as static files in the publicsite
- Generated by `python3 -m codifide capability` and `python3 -m codifide capability --cbor`
- Served at `codifide.com/capability.json` and `codifide.com/capability.cbor`
- Vercel route added to `vercel.json`

**REQ-2b: Agent cookbook**
Structure:
```
docs/AGENT_COOKBOOK.md
  - Top 10 failure modes from v1.0 reviews
  - Each entry: intention → wrong guess → correct form → working example
  - Covers: arithmetic, string ops, control flow, belief dispatch, effects, refusal
```

**REQ-2c: Feedback template**
```
dispatches/feedback/TEMPLATE.md
  - Model name and version
  - Task attempted
  - What failed (exact error)
  - What was tried
  - What worked
  - Suggestions
```

**REQ-2d: Agent quickstart command**
New CLI subcommand: `python3 -m codifide agent-quickstart`
- Checks Python version
- Runs the test suite
- Writes `examples/quickstart.cod` (a hello-world with intent, effects, belief)
- Runs it
- Prints the content hash
- Prints "You are ready to write Codifide."

### Track 3: v2.0 Roadmap

Deferred until Track 1 findings are in. The design for v2.0 will be a separate spec.

Placeholder priorities based on v1.0 review findings:
1. **RPC API** — agents need to speak canonical form without text parsing
2. **Effect inference** — reduces boilerplate for pure functions
3. **Inline conditional** — already shipped in v1.0 ✅
4. **Rust interpreter** — performance for production agent use

---

## Dispatch Discipline

Every session in this initiative:
- Opens with a Quill readout naming the session goal
- Closes with a paired Glyph YAML
- Sable audits at each track boundary (end of Track 1, end of Track 2)
- `dispatch-check` runs at session close
