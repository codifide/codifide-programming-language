## Finding CLO-1 — Bind-before-when (P2)

**Problem:** A `cand` block with a bind (`<-`) followed by a `when` guard
looks syntactically plausible but fails at runtime. Guards execute before
candidate bodies — the bound name doesn't exist yet when the guard runs.
The error was `unbound name: 'label'` with no explanation of why.

**Fix applied:**

1. **Runtime hint** (`codifide/runtime/interpreter.py`) — `_unbound_name_message`
   now appends a bind-before-when explanation for any unbound plain identifier,
   pointing at the fix (move bind into body, use `if/then/else`). The
   `_unknown_callable_message` function got the same treatment for the case
   where the name is parsed as a call target.

2. **`docs/AGENT_QUICKREF.md`** — added bind-before-when to the "surface rules
   that surprised agents" section with a broken example, the error, and the fix.

3. **`docs/GPT4O_PROMPT.md`** — same rule added to the prompt's surface rules
   section.

**Verified:** The hint fires correctly on a test file. 289 tests pass, 0
regressions.

## Assessment

Three sessions, four findings applied. The docs and error messages are now
materially better than when Track 1 started. The bind-before-when hint is
the most valuable runtime improvement — it turns a confusing `unbound name`
error into a direct explanation of the execution order.

What I'm not yet sure of: whether there are other guard-vs-body execution
order surprises. The `when` guard runs pure and before the body — any
expression that depends on body-side state will fail the same way. A Sable
audit of the guard evaluation model would be worth filing as part of T1-6.
