# Noema session-end — polish, hardening, and the story that holds together

*By Quill. 10 May 2026.*

One session. What started as "spin up Rust and see if it's real" turned
into a language with a specification, two implementations that agree,
a content-addressed symbol store, content-addressed imports, indices for
discovery, three personas that hold the project accountable, and a body
of documentation that reflects what actually exists rather than what
was once planned.

This is the session-end readout. Short, because the dispatches that
preceded it covered the meat.

## What the last turn added

**Better error messages that walk the intent graph.** Roadmap v0.1 item.
When a contract or effect violation fires, the error now names every
frame on the call path and its intent, innermost-first. Three levels
deep in a synthetic example, the message reads:

```
ContractViolation: 'inner' failed postcondition `lt(result, 0)`. The
function exists because: 'asserts something false'
  called from:
    outer: 'top-level orchestrator'
      middle: 'computes a thing'
        inner: 'asserts something false'
```

The agent reading that error knows not just what failed but *why the
failing call was being made*. Intent is first-class; a good error
message respects that.

**A richer pure-primitive library.** Math (`abs`, `min`, `max`, `pow`,
`floor`, `ceil`, `round`), collections (`min_of`, `max_of`, `sum`,
`reverse`, `append`, `contains_item`), strings (`upper`, `lower`,
`trim`, `starts_with`, `ends_with`, `replace`, `split`, `join`). All
pure. All typed. Empty-list aggregates surface as `PrimitiveError`.
Nothing host-exception leaks.

**`noema store verify`.** Closes Sable's P3-1 from the index audit.
Hand it a stored module's identity and it walks the module's imports,
confirms every pointee is present in the store, and exits non-zero with
a specific list if any are missing. Opt-in because doing this at parse
time would be a DoS vector; doing it at audit or publish time is cheap
and right.

**Parser hardening.** The fuzz harness — 30 hand-curated adversarial
inputs and 200 structurally generated ones — found two real P1 bugs.
`LexError` was escaping through `_safe_parse_expr` for unclosed
strings, non-ASCII outside strings, and stray operator sequences.
Deep parenthesis nesting (500+) blew the Python stack and surfaced as
`RecursionError`. Both fixed: the wrapper now catches `LexError`,
and the expression parser enforces a 256-deep paren bound that raises
a typed `ParseError` instead.

**Store concurrency test.** Eight processes writing the same symbol
simultaneously all agree on the identity, exactly one object lands on
disk, no `.noema-*.tmp` files leak. Eight processes writing distinct
symbols produce eight distinct objects. The atomic-rename write path
holds up under real contention.

**Documentation.** `README.md` rewritten to reflect current reality.
`docs/LANGUAGE.md` updated with imports, indices, shadowing, typed
errors, recursion limit, store CLI. `docs/TUTORIAL.md` new — a
thirty-minute walk-through from first program to publishing an index
and consuming through it. `docs/ARCHITECTURE.md` new — the builder's
map, layered structure, two-implementation strategy, test topology,
file map for contributors. `GETTING_STARTED.md` updated.

76 Python tests pass. 6 Rust canonical tests pass. `cargo audit` was
clean at last run; re-running at every gate is the discipline going
forward.

## What I'm not yet sure of

The biggest open item is the one Quill keeps circling back to:
**garbage collection for the store**. Monotonic growth is fine for a
scratch session; a production agent that produces variants constantly
would hit a disk-space wall within weeks. A store GC that preserves
reachable identities from a declared root set is the right shape, but
it's a project, not a patch.

**CBOR wire form** is still future work. JSON is verbose; CBOR would
be meaningfully smaller and keep cross-implementation byte-equality.
The conformance test generalizes cleanly — same bytes, different
serializer.

**Cost-based dispatch.** Today candidate selection is by guard order.
A real agent-first dispatcher would pick among candidates using
latency, memory, and trust budgets. The canonical form already carries
enough metadata; the dispatcher itself is the work.

**The Rust interpreter.** Still deferred, by design. Semantics keep
shifting; porting them now doubles the cost of every subsequent
change. v0.3 remains the right moment.

**What Sable would find on the next pass.** Every new capability
increases the attack surface. The next audit should look at the CLI's
filesystem assumptions (symlinks, concurrent growth, path traversal)
and at the Rust crate under its own fuzz. The parser fuzz corpus can
be ported to Rust when the Rust side gains a parser.

Session state: `sessions/2026-05-10.md` captures the cumulative context
so the next session can resume without reading the whole dispatch
history. The dispatches themselves are the full record; read them in
filename order to retrace the path.

Noema is a prototype that knows what it is. That is the thing that
changed today.
