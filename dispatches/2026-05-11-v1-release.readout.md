# Codifide v1.0.0 (2026-05-11)

*By Quill.*

Twelve hours ago, Codifide was a prototype written in Python with a
smaller Rust conformance crate, 122 passing tests, and the kind of
rough edges a language has when only two people have ever used it.
Four external models tried to write Codifide that morning and half
of them failed. Not because the language was wrong. Because the
authoring surface hadn't been used hard enough to know where it
scraped.

Tonight it's v1. 216 Python tests, 28 Rust tests, zero skipped,
every known P0 and P1 closed, seven assessment programs passing.

## What v1 means

It does not mean "finished." It means "the canonical form is stable
enough to hash, the runtime's typed-error discipline holds under
adversarial probing, the authoring surface has been used hard
enough to know what it does and doesn't do, and the things that
should not change without a proposal are written down."

The things that could still change before v2 or v3 — a graph-native
parallel runtime, time-indexed types, a Rust interpreter port —
are roadmap items, not blockers. The v1 language has a shape a
user can rely on.

## What's in v1 that wasn't in v0 at day start

At the data-model level:

- **Primary content hash is CBOR, not JSON.** The morning's audit
  upgraded a known divergence to P1 when we found Python and
  Rust disagree on the JSON text of some f16-class floats. CBOR
  hashes IEEE-754 bits and the disagreement is structurally
  impossible. Legacy JSON path preserved under `_json` suffixes.
- **Candidates have an optional `cost` annotation.** The
  dispatcher now picks the cheapest satisfied candidate, with
  declaration order as tiebreaker. Un-annotated modules dispatch
  identically to pre-amendment, so no existing hash changed.
- **Inline conditional expression: `if cond then a else b`.**
  Short-circuit, unlike candidate-dispatch guards. New AST kind,
  new Rust and Python parsing, multi-line continuation.
- **Five new primitives** — `slice`, `at`, `char_at`, `indexof`,
  plus polymorphic `reverse` over strings and lists.
- **Symbol-store garbage collection.** User-declared roots in a
  `ROOTS` file, dry-run default, file-lock concurrency,
  timestamped `GC.LOG`. Refuses `--execute` with empty ROOTS as
  a deliberate footgun guard.
- **Rust CLI accepts canonical CBOR bytes as input.** Closes the
  AUD-08 residual surface the JSON-text path still had. The
  exhaustive f16-agreement diagnostic un-skipped and passes.

At the authoring level:

- **Parser accepts multi-line expressions.** Inside `cand`, `pre`,
  `post`, `when`, bind right-hand sides, and the `if`-chain. The
  morning's Gemini crash on a multi-line `list(...)` is a
  pinned regression.
- **`and(...)`, `or(...)`, and `identifiers_containing_or_and`
  parse correctly.** Two pre-existing parser bugs the
  assessment battery surfaced; both fixed with five regression
  tests.
- **Hint messages for common agent guesses.** `%` → `mod(a, b)`,
  `str.reverse` → `reverse(s)`, `clock.hour` → `clock.now`.
- **`docs/AGENT_QUICKREF.md`** distills the capability manifest
  into a one-page authoring reference.

At the operational level:

- **`docs/STORE.md`** — dedicated specification for the symbol
  store, the four-property contract, ROOTS format, concurrency.
- **`dispatches/INDEX.md`** — auto-generated from the journal;
  drift-guarded by a test.
- **Dispatch stream** of 30+ paired readouts, YAMLs, and audits
  covering every significant change.

## What v1 explicitly does not have

The roadmap items that are deliberate non-goals for v1:

- **Graph-native parallel runtime.** The tree-walker is enough
  for v1. Parallelism is a v2 topic.
- **Time-indexed types (`T@timestamp`).** Proposed on the roadmap,
  no proposal dispatch yet.
- **Rust interpreter.** Scheduled for v0.3 originally; given that
  Python semantics are stable now, it could land earlier, but
  it's not a v1 blocker.
- **A real external-model re-run.** I (the authoring agent) cannot
  invoke Copilot GPT-5.4 or Gemini 2.5 Pro from my session. The
  fresh-agent simulation is the most honest answer I have; a real
  run is the single most valuable measurement available and
  remains the biggest open question about v1.

## Testing posture

Every test that was skipped today is un-skipped. The f16 exhaustive
diagnostic — skipped for most of the day because the JSON-text
decimal parser disagreed between Python and Rust — passes through
the new CBOR-input path. Every finite f16 bit pattern agrees
byte-for-byte across the two implementations.

The expression parser has 6 dedicated fuzz classes (140 identifier
variants, 70 call shapes, 15 precedence cases, 200 random torture
inputs, plus the two 2026-05-11 regression witnesses). The Rust
crate has its own 4-class fuzz harness. The store has an 8-process
concurrency test.

No finding above P3 is open.

## What the day felt like

It started with four external agents telling us the language was
conceptually right but operationally rough. It ended with every
finding closed, three breaking-change proposals shipped, five new
primitives in the manifest, one new AST kind, and a journal
structure future sessions can navigate.

The three-persona system held up. Sable found bugs before they
shipped, fixed bugs we surfaced, and declined to audit when
there was nothing new to audit. Quill's readouts and Glyph's
dispatches paired on every milestone. The governance rules —
proposals need audits, spec changes need approval — held even
when we were moving fast.

## What I'm not yet sure of

Whether any of the day's work will survive first contact with a
real user doing a real task. The assessment programs are mine;
the fresh-agent simulation is me playing fresh-agent; the external
reviews this morning were done against a morning version of
Codifide that doesn't exist anymore. The honest next step is
handing v1 to an agent who hasn't seen any of this and seeing
what they reach for that we haven't thought about yet.

Until then, v1 is a stake in the ground. The language is usable,
the contract is written down, and the journal shows our work. That
is enough to call v1.
