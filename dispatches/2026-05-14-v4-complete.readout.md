## What happened

The user asked whether Codifide is usable in the wild. The honest answer was
"not yet" — four structural gaps between research prototype and usable tool.
That answer became the v4.0 requirements. Three of four shipped in this session.

---

## What shipped

### V4-1 — Runtime type enforcement

`sig` declarations are now enforced at every user-function call boundary.
`TypeViolation` is raised when a value does not match the declared type.

The type hierarchy: `Number` accepts `Int` and `Float`. `String` accepts
`Label`. `Any` accepts everything. Untagged (`Any`) actual values pass all
checks — type enforcement is best-effort, not a full static type system.

This was the right call. A full static type system would require a type
inference pass that doesn't exist yet. Best-effort runtime checking catches
the obvious mistakes (passing a string where an int is declared) without
breaking programs that rely on the existing `Any`-typed value flow.

20 new tests in `tests/test_type_enforcement.py`. All 450 tests passing.

### V4-2 — Standard library

Four new primitive groups:

**File I/O** (`{io.read}`, `{io.write}`): `io.read`, `io.write`, `io.exists`.
Path traversal defense built in. 16 MiB size limit. Effect-enforced.

**HTTP client** (`{http.fetch}`): `http.get`, `http.post`. HTTPS-only.
30-second timeout. 16 MiB response limit. Non-2xx raises `PrimitiveError`.

**JSON** (pure): `json.parse`, `json.encode`. No effect declaration needed.
JSON objects become dicts accessible via `at(obj, "key")`.

**Date arithmetic**: `clock.today` (effect: `{clock.read}`), `clock.parse`,
`clock.add_days`, `clock.format` (all pure).

44 new tests in `tests/test_stdlib.py`. Capability manifest updated.
`docs/AGENT_QUICKREF.md` updated with new primitive groups and examples.

### V4-3 — Public registry (operational)

`docs/REGISTRY.md` documents the publish-and-resolve workflow. The five
canonical pipeline symbols are published to the public registry.

### V4-4 — Network-safe server — deferred

Bearer token auth and TLS for the RPC server were scoped but deferred.
The public registry must demonstrate multi-machine use cases before the
security investment is justified. No adoption evidence yet.

## What I'm not yet sure of

Whether the best-effort type checking is the right long-term design. It
catches obvious mistakes but misses cases where a value is tagged `Any`
but holds the wrong type. A full static type inference pass would catch
those — but that's a significant design investment. The current approach
is honest about its limits and documented as such.
