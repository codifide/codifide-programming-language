## T2-4 — Feedback dispatch template

`dispatches/feedback/TEMPLATE.md` written. Structured template for any agent
to report observations, failures, and suggestions. Fields: header (model,
manifest hash, task, session type), what failed (code + error + fix attempt),
surprises, primitives missed, what worked, suggestions, programs completed
table, one-sentence assessment.

The template is in Quill/Glyph format — it produces a human-readable readout
that Sable can review. Referenced in `docs/FOR_AGENTS.md` and `docs/AGENT_COOKBOOK.md`.

## T2-5 — `python3 -m codifide agent-quickstart`

New CLI subcommand implemented in `codifide/__main__.py`. Six steps:

1. Check Python version (3.9+ required)
2. Run the test suite — confirms interpreter is healthy
3. Write `examples/quickstart.cod` — hello-world with intent, effects, belief,
   and content addressing
4. Run it
5. Print the content hash of every symbol
6. Print "You are ready to write Codifide." with next steps

The quickstart program demonstrates all four things that make Codifide
different: intent, effects, belief dispatch, and content addressing — in
under 20 lines.

## Assessment

T2-4 through T2-6 complete in one pass. The feedback template is the right
shape — short enough that an agent will fill it in, structured enough that
Sable can review it. The quickstart command is the right entry point — it
validates the environment, demonstrates the language, and leaves the agent
with a working program and its content hash.

What I'm not yet sure of: whether the quickstart should also fetch and display
the manifest hash from `codifide.com/capability.json` to confirm the agent is
working against the current language version. That would close the loop between
the local install and the public endpoint.
