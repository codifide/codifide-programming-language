**Date:** 2026-05-14

## What happened

Sable audited the RPC API surface. Five findings. Two P2s applied
immediately. Three P3s applied or deferred.

---

## Findings and resolutions

**AUD-RPC-01 (P2) — Negative Content-Length** — Fixed. `_read_body` now
returns `None` for negative values before attempting any read. Test added.

**AUD-RPC-02 (P2) — No socket timeout** — Fixed. `make_server` now sets
a 30-second socket timeout on the server socket. Prevents slow-loris
resource exhaustion.

**AUD-RPC-03 (P3) — Unused imports** — Fixed. `hashlib` and `os` removed
from `server.py`.

**AUD-RPC-04 (P3) — `/imports` resolves one level only, not documented** —
Fixed in docs. `docs/RPC_API.md` now says "direct imports only" and
instructs callers to recurse for the full closure. The implementation is
correct; the spec was misleading.

**AUD-RPC-05 (P3) — `threading` unused directly** — Accepted as-is.
`threading` is used indirectly via `ThreadingHTTPServer`; removing it
would make the import intent less clear, not more.

---

## What I'm not yet sure of

Whether the 30-second socket timeout interacts correctly with the
`ThreadingHTTPServer`'s connection handling under Python 3.9. The timeout
is set on the server socket (the listening socket), not on accepted
connection sockets. This may not actually bound per-request read time.
A follow-up probe is needed if the server is ever exposed over a network.
