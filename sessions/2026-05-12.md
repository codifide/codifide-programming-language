## What Shape A means

**Thesis.** Codifide's correctness claim is verified. Now its
performance and concurrency claims need to be.

**Scope (from `dispatches/2026-05-11-v2-thinking.readout.md`):**

1. **Rust interpreter port.** The whole evaluator, not just the
   canonical form. Python reference stays as the spec; Rust becomes
   the production runtime. Expected 10-100× speedup on realistic
   programs.

2. **Graph-native parallel evaluator.** Today's tree-walker evaluates
   sequentially even when the dataflow graph has independent branches.
   A real implementation parallelizes at the node level. This is in
   the original seven design principles ("parallelism is default;
   sequencing is declared") and v1 does not deliver on it.

3. **Effect-scoped concurrency model.** If two branches share an
   `io.stdout` effect, they must serialize; if their effect sets are
   disjoint, they can run in parallel. The effect algebra already
   tells us what is safe; the runtime just has to honor it.

4. **Benchmarks as first-class citizens.** Every example gets a
   performance fixture. Regressions are regressions.

**Estimated scope:** 2-3 sessions.

**Biggest risk:** porting a moving target. Mitigation: the spec is
now written, audited, and stable. The target is less moving than
when we first deferred this work.

## What the Rust interpreter needs to implement

The Python interpreter in `codifide/runtime/interpreter.py` is the
reference. The Rust port must match it on:

- Candidate dispatch (including cost-based selection).
- Effect enforcement (primitive-call half + transitive half).
- Contract evaluation (pre/post/guard with empty effect budget).
- Belief dispatch (`believe` blocks).
- Inline conditional (`if ... then ... else`, short-circuit).
- First-class refusal (`bottom`, `RefusalError`).
- Recursion limit (default 64, typed `RecursionLimitError`).
- All eight typed error classes.
- Import resolution through the symbol store.

The graph-native parallel evaluator is a second step after the
tree-walker port passes all tests. Don't parallelize until the
sequential semantics are confirmed correct.
